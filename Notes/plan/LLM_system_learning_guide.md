## #1 底层原理（KV Cache / RoPE / Attention/GPU的底层原理）

### 📘 学什么？

-   Transformer Attention 公式\
-   Q/K/V 的真正意义\
-   KV Cache 如何加速\
-   RoPE / 旋转位置编码如何让模型"知道位置"\
-   解码阶段为什么是一步一步生成\
-   为什么高并发推理瓶颈在 KV Cache（不是算力）

### 📚 从哪里学？

-   **直接读源码（强烈推荐 PyTorch + vLLM + Qwen）**
-   Hugging Face `transformers` 里 `modeling_xxx.py`
-   FlashAttention 论文 + 源码 + 稀疏注意力（你已经懂了）

### 🔍 带着哪些角度读？

-   注意力是如何减少复杂度的？\
-   为什么 KV Cache 能省时间？\
-   为什么 RoPE 会出现 "位置偏移/位置外推能力"？\
-   你的推理服务器显存到底被什么占满？\
-   为什么你 batch 越大速度越快？

## #2 RAG 工程问题（长文切分、召回、OOV）

### 📘 学什么？

-   文档切分策略（chunking）与"语义断点"\
-   Embedding 召回的 recall@k\
-   OOV（超出词表）如何导致召回失效\
-   向量库（FAISS / Milvus）如何优化\
-   Re-ranking 解决长文错误匹配

### 📚 从哪里学？

-   **大厂博客：**
    -   OpenAI Cookbook\
    -   阿里通义 RAG 文档\
    -   百度文心 RAG 教程\
    -   微软 Azure AI RAG 指南
-   **实践调试**
    -   LangChain\
    -   LlamaIndex\
    -   RAGFlow

### 🔍 带着哪些角度读？

-   你切分文档的策略是否会打碎语义？\
-   你召回的向量是否"噪声比信号多"？\
-   OOV 单词会导致 embedding 失效吗？\
-   模型为什么根据错误段落回答？\
-   Re-ranking 是否比你换模型更重要？

### 📚 从哪里学？

-   **大模型开源社区经验文档：**

-   为什么 validate loss 很低但实际问答很差？\
-   训练数据哪部分最影响最终风格？

## #5 阅读资料时的问题框架（10 大核心提问）

1.  **它解决什么问题（Problem）？**\
2.  **为什么之前的方法不行（Pain Point）？**\
3.  **核心思想是什么（Key Idea）？**\
4.  **公式/算法的关键点是什么（Mechanism）？**\
5.  **有什么 trade-off？牺牲了什么？**\
6.  **瓶颈是什么？适用于什么场景，不适用于什么？**\
7.  **跟我目前的需求有什么联系？**\
8.  **是否能复现 / 是否有源码？**\
9.  **这个方法对推理速度/显存有什么影响？**\
10. **如果我要实现，最小可行代码是什么？**

## #6 不同模块应该带着怎样的角度阅读？

### 🔷 底层（Attention、RoPE、KV Cache）

**带数学 + 工程视角**\
- 公式如何转成 kernel？\
- 内存访问 pattern 是怎样？

### 🔷 RAG

**带信息检索 IR + 软件工程视角**\
- 如何减少噪声？\
- 如何提高召回准确率？\
- 如何自动化清洗文本？

### 🔷 量化 / 部署

**带 GPU 工程 + 性能优化视角**\
- 你的 GPU 算得慢是因为哪里？\
- kernel 启动是否产生 overhead？

### 🔷 微调

**带训练动态 + 数据工程视角**\
- 数据是否干净？\
- loss 是否在振荡？\
- LoRA 的结构是否合适？

## #7 学习方法总结

1.  **底层原理一定要看源码，不要只看博客**
2.  **RAG 优化靠"实验 + 数据分析"**
3.  **量化 / 部署靠读 NVIDIA 文档**
4.  **微调要看开源组织的"踩坑总结"**
5.  **读任何内容都带着"问题框架"（上面 10 条）**
6.  **边学边做小实验，不能只看不练**
7.  **遇到难点，把它画成图（Attention、RoPE 特别有效）**
